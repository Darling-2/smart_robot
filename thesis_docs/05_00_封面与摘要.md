## 摘要

随着人工智能、嵌入式计算与机器人技术的深度融合，以移动机器人为载体的自主巡检系统在智能建筑、公共安全和工业设施管理领域展现出广阔的应用前景。传统人工巡检方式存在劳动强度大、覆盖频率低、记录不规范等固有缺陷，在需要全天候监测的室内场景中尤为突出。然而，现有自主巡检系统普遍依赖高性能计算平台或单一机器人操作系统版本，面向低成本嵌入式平台的全栈异构系统集成方案研究仍存在较大空白。

本文研究并实现了一套面向室内走廊场景的移动机器人自主巡检系统。以 NanoRobot 差速驱动底盘为载体，采用**异构双处理器架构**：**香橙派3LTS运行 ROS1 Noetic**，负责差速底盘运动控制（串口驱动 STM32 控制板）、EAI 单线激光雷达数据采集，以及二维建图定位与自主导航；**树莓派5（4GB）运行 ROS2 Jazzy**，负责巡检任务调度与 YOLOv8n 视觉检测推理。两板通过百兆以太网直连，跨板通信采用 **ZeroMQ（ZMQ）** 在应用层传输最小业务消息：树莓派下发导航目标，香橙派回传导航结果，避免在树莓派侧引入 ROS1 依赖并降低跨板带宽与耦合度。

在地图构建方面，系统在香橙派 ROS1 侧部署 GMapping 完成二维栅格地图构建（分辨率 0.05m），并在巡检阶段切换至 AMCL 实现粒子滤波定位。在自主导航方面，香橙派侧使用 ROS1 move_base 导航框架完成全局路径规划与局部避障控制（可配置 DWA/TEB 等局部规划器），树莓派侧按巡检点序列逐一发送目标并等待结果回执。在视觉感知方面，针对室内设施状态检测任务构建自定义数据集（≥600张标注图像），通过迁移学习微调 YOLOv8n 模型，在树莓派5上基于 ultralytics 框架直接加载 PyTorch 权重进行 CPU 推理，将检测频率控制在约 5 FPS，mAP@0.5 达到 **XX.X%**。最终，系统在室内走廊场景完成多点顺序自主巡检任务验证，综合巡检任务完成率达 **XX%**，验证了异构 ROS 架构下低成本嵌入式平台执行完整自主巡检任务的工程可行性。

## Abstract

With the deep integration of artificial intelligence, embedded computing, and robotics, autonomous mobile robot inspection systems have demonstrated broad application prospects in smart building management, public safety, and industrial facility monitoring. Traditional manual inspection methods suffer from inherent limitations including high labor intensity, low coverage frequency, and non-standardized records, which are particularly pronounced in indoor environments requiring around-the-clock monitoring. However, existing autonomous inspection systems typically rely on high-performance computing platforms or a single version of Robot Operating System (ROS), leaving a significant research gap in full-stack heterogeneous system integration for low-cost embedded platforms.

This paper presents the design and implementation of an autonomous mobile robot inspection system for indoor corridor scenarios. Built upon a NanoRobot differential-drive chassis, the system adopts a **heterogeneous dual-processor architecture**: the **Orange Pi 3LTS runs ROS1 Noetic** for chassis motion control (serial-port-driven STM32), single-line LiDAR acquisition, and 2D mapping/localization/navigation; the **Raspberry Pi 5 (4GB) runs ROS2 Jazzy** for patrol-task orchestration and YOLOv8n visual detection inference. The two boards are connected via 100Mbps Ethernet, and inter-board communication is implemented with **ZeroMQ (ZMQ)** at the application layer to exchange minimal business messages (navigation goals and results), avoiding ROS1 dependencies on the ROS2 side.

For mapping, the system deploys GMapping on the ROS1 side to build 2D occupancy grid maps (0.05m resolution), and switches to AMCL for particle-filter localization during patrol runs. For navigation, ROS1 move_base is used for global planning and local obstacle avoidance (with configurable local planners such as DWA/TEB), while the ROS2 side sends waypoint goals and waits for result acknowledgements. For visual perception, a custom dataset (≥600 labeled images) is constructed for indoor facility state detection, YOLOv8n is fine-tuned via transfer learning, and the model is deployed on the Raspberry Pi 5 via the ultralytics framework with direct PyTorch CPU inference, throttled to approximately 5 FPS, achieving mAP@0.5 of **XX.X%**. The complete system is validated in indoor corridor scenarios with multi-point sequential patrol tasks, achieving an overall task success rate of **XX%**.
