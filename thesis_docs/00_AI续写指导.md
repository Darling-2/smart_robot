## 第2节 硬件与系统架构（已确认）

### 2.1 双板架构

| 板卡 | 系统 | 功能 | IP |
|:---|:---|:---|:---:|
| 香橙派3LTS（RK3566，2GB RAM） | Ubuntu 20.04 + ROS1 Noetic | 底盘控制 + 雷达 + SLAM/定位 + move_base 导航闭环 | 192.168.10.1 |
| 树莓派5（BCM2712，4GB RAM） | Ubuntu 24.04 + ROS2 Jazzy | 相机 + YOLO 视觉检测 + 巡检任务调度 | 192.168.10.2 |

### 2.2 外设连接

| 外设 | 连接板卡 | ROS话题 |
|:---|:---|:---|
| STM32底盘控制板（差速驱动） | 香橙派（USB串口 `/dev/ttyUSB0`） | 发布 `/odom`，订阅 `/cmd_vel` |
| EAI YDLidar（单线，10Hz） | 香橙派（USB串口） | 发布 `/scan` |
| 720p USB摄像头 | 树莓派5（USB） | 发布 `/image_raw` |

### 2.3 网络连接

- 两板通过**百兆以太网网线直连**（非经由路由器）
- 静态IP：香橙派 `192.168.10.1`，树莓派5 `192.168.10.2`
- `ping` RTT 约为亚毫秒~毫秒级（以实测为准）

### 2.4 工作空间

- 香橙派 ROS1 工作空间：`bingda_ros1_noetic-nano`
- 树莓派5 ROS2 工作空间：`smart_robot-main`（本仓库，含 `robot_vision_ros2`、`robot_navigation_ros2` 等包）

## 第3节 关键技术参数与口径（以配置文件为准）

本节用于约束“哪些内容可以写死、哪些必须留占位符”。续写时严禁凭空编造参数数值；所有数值必须来自工作空间配置文件或实测数据。

### 3.1 跨板通信（ZMQ业务消息）

- 仅跨板传输两类业务消息：`nav_goal`（ROS2→ROS1）与 `nav_result`（ROS1→ROS2）
- 推荐端口约定（如已在代码中使用）：`5555`（goal），`5556`（result）（如与实物不同，以代码为准）
- 字段建议（JSON）：
       - `nav_goal`: `goal_id, x, y, yaw, frame_id, stamp`
       - `nav_result`: `goal_id, status, text, stamp`

### 3.2 ROS1 导航（主线）

- 采用 `slam_gmapping` 建图，`amcl` 定位，`move_base` 导航闭环
- 相关参数文件集中在：`bingda_ros1_noetic-nano/robot_navigation/param/NanoRobot/`（如 `dwa_local_planner_params.yaml`、`costmap_common_params.yaml` 等）

### 3.3 YOLOv8n 训练与部署参数（来自 `yolo_training/`）

- 模型：YOLOv8n（nano），预训练权重 `yolov8n.pt`（COCO）
- 训练配置：`epochs=120`（当前），`freeze=10`（前10层），`batch=16`，`lr0=0.01`
- 当前训练结果（435张样本）：mAP@0.5 = 80.6%，Precision = 85.2%，Recall = 77.1%
- 目标：扩充至 ≥600张后，mAP@0.5 ≥ 85%
- 部署：ONNX + ONNXRuntime，树莓派5约 **54ms/帧（≈18 FPS）**，内存 ~280MB
