# 第6章 基于YOLOv8n的目标状态检测

## 6.1 目标检测技术发展综述

目标检测（Object Detection）是计算机视觉领域中最具挑战性的基础任务之一，要求算法在图像中同时完成目标的**定位**（预测边界框坐标 $[x_{\min}, y_{\min}, x_{\max}, y_{\max}]$）与**分类**（判断目标所属类别及置信度），是图像分类、语义分割等其他视觉任务的重要基础。

深度学习时代之前，目标检测主要依赖手工设计特征（如 HOG、SIFT）结合滑动窗口搜索，精度与速度均受限于手工特征的表达能力。自2012年 AlexNet [3] 在 ImageNet 竞赛上以大幅优势夺冠，卷积神经网络（CNN）迅速成为目标检测的核心特征提取器。

### 6.1.1 两阶段检测器

以 Faster R-CNN [20] 为代表的**两阶段（Two-Stage）检测器**将检测流程分为"区域提议（Region Proposal）"与"区域分类回归"两步：区域提议网络（RPN）首先生成若干候选框，再由检测头对每个候选框进行精细的分类和边界框回归。两阶段检测器在 COCO 等标准基准上长期保持精度第一，但串行流程使其推理速度受限——典型的 Faster R-CNN（ResNet-50 主干）在 V100 GPU 上约 7 FPS，在 ARM 嵌入式平台上约 0.5 FPS，远不满足机器人实时应用需求。

### 6.1.2 YOLO 系列单阶段检测器演进

**单阶段（One-Stage）检测器**将区域提议与分类回归合并为一个步骤，直接从特征图上密集预测所有候选检测框，以精度换速度。Redmon 等人于2016年提出的 **YOLOv1** [4] 将图像划分为 $S \times S$ 栅格，每格直接预测 $B$ 个边界框及 $C$ 类别概率，将检测问题统一为单次前向推理，首次实现了45 FPS 的实时检测，开创了单阶段检测器的先河。

此后 YOLO 系列经历了持续的性能迭代：
- **YOLOv3** [5]（2018）引入多尺度预测（借鉴 Lin 等人 [21] 的 FPN），通过 3 个不同分辨率特征图分别检测大/中/小目标，显著提升小目标检测能力；
- **YOLOv4** [13]（2020）系统集成 CSPNet 主干、PANet 颈部、DIoU 损失、Mosaic 数据增强等一系列"训练技巧"，建立了兼顾速度与精度的综合优化范式，COCO AP 达到43.5%@65 FPS（V100）；
- **YOLOv5** [24]（2020，Ultralytics）基于 PyTorch 进行全面工程化重构，引入 AutoAnchor 自适应锚框，以友好的 Python API 极大降低了部署门槛，成为工业界最广泛使用的检测框架之一；
- **YOLOv6** [14]（2022，美团）和 **YOLOv7** [15]（2022，WongKinYiu）分别在面向工业部署的模型量化友好性和模型缩放策略方面提出改进；
- **YOLOv8** [6]（2023，Ultralytics）在 YOLOv5 架构基础上引入**无锚框（Anchor-Free）解耦检测头**与 **C2f（Cross Stage Partial with 2 bottlenecks）主干模块**，以 Distribution Focal Loss（DFL）提升定位精度，并统一了检测、分割、关键点估计等多种任务的接口，成为目前综合表现最好的单阶段检测框架之一。

Terven 和 Cordova-Esparza [16] 于2023年对 YOLOv1 至 YOLOv8 及 YOLO-NAS 的架构演进进行了系统综述，指出 YOLO 系列已成为机器人、自动驾驶和安防监控领域最广泛使用的实时目标检测框架。

### 6.1.3 本系统选用 YOLOv8n 的理由

本系统最终选用 **YOLOv8n**（Nano 版本）作为视觉检测主干，主要理由如下：

1. **极低参数量**：YOLOv8n 仅约 **3.2M 参数**、**8.7 GFLOPs**，是 YOLOv8 系列中最轻量的版本，在树莓派5（BCM2712，无 GPU）上基于 ultralytics 框架直接加载 PyTorch 权重可实现约 **3–5 FPS** 的检测频率，配合 `max_fps` 参数节流即可满足巡检场景的实时性要求；
2. **高精度迁移能力**：基于 COCO 预训练权重的迁移学习，即使自定义数据集规模较小（≥600张），也能快速收敛到较高的 mAP；
3. **工程化友好**：Ultralytics 提供完整的 Python 训练 API、推理接口与多格式导出工具链，在树莓派5上可直接使用 `model.predict()` 进行推理，部署流程简洁可靠；
4. **内存占用适中**：ultralytics PyTorch CPU 推理时内存占用约 **280–300 MB**，与树莓派5侧的巡检调度与 ROS2 通信组件同时运行不产生内存瓶颈；后续可通过 ONNX 导出进一步优化推理速度。

## 6.2 YOLOv8 网络架构

YOLOv8 的整体架构遵循现代目标检测网络的**主干（Backbone）+ 颈部（Neck）+ 检测头（Head）**三段式设计，各部分功能与改进如下所述。

### 6.2.1 C2f 主干（Backbone）

YOLOv8 的主干以 **C2f 模块**（Cross Stage Partial with 2 Bottlenecks）替代 YOLOv5 的 C3 模块，其核心设计思想来源于 CSPNet [25]——将输入特征图按通道分为两路：一路经过 $n$ 个 Bottleneck 进行深层特征变换，另一路直接跨越（skip connection），两路在最后拼接后经 $1\times 1$ 卷积融合。

与 C3 的区别在于：C2f 将所有 Bottleneck 的中间特征均引入最终拼接（类似 DenseNet 的密集连接），而非仅连接最后一个 Bottleneck 的输出，有效改善了梯度流（Gradient Flow），使模型在更少的参数下保持更丰富的特征表示能力。

YOLOv8n 主干的具体层次结构（以输入 $640\times 640$ 为例）：

```
Input (640×640×3)
  → Conv (3×3, s=2) → 320×320×16
  → Conv (3×3, s=2) → 160×160×32
  → C2f(n=1)        → 160×160×32
  → Conv (3×3, s=2) → 80×80×64
  → C2f(n=2)        → 80×80×64
  → Conv (3×3, s=2) → 40×40×128
  → C2f(n=2)        → 40×40×128
  → Conv (3×3, s=2) → 20×20×256
  → C2f(n=1)        → 20×20×256
  → SPPF             → 20×20×256  （多尺度空间金字塔池化）
```

主干末端的 **SPPF（Spatial Pyramid Pooling-Fast）** 模块通过三次连续的 $5\times 5$ 最大池化（等效于 $5/9/13$ 感受野）扩大特征的感受野，对大目标和全局上下文理解能力有重要贡献。

### 6.2.2 PANet 颈部（Neck）

YOLOv8 颈部采用 **PANet（Path Aggregation Network）** [26] 结构，在 FPN 自顶向下的特征金字塔基础上，增加了**自底向上的特征路径增强（Bottom-up Path Augmentation）**：

```
                    ↑ FPN 自顶向下
P5 (20×20) ─────────────────────────── P5'
   ↓ 上采样                              ↑
P4 (40×40) ─── C2f ─────── (上采样) ─── P4'
   ↓ 上采样       ↑                      ↑
P3 (80×80) ─── C2f ─── (下采样) ─── (C2f) P3'
              ↓ PANet 自底向上 ↑
```

FPN 路径保证了高层语义特征能够自顶向下传递到小目标检测分辨率（P3，80×80）；PANet 路径则保证了精确的定位信息（来自浅层）能够自底向上传递到大目标检测分辨率（P5，20×20）。两路路径的信息融合使 YOLOv8 对不同尺度目标均具有良好的检测能力。

本系统检测目标（室内门，典型面积约占图像的20%~60%）主要属于中大型目标，P4（40×40）和 P5（20×20）特征图贡献最大。

### 6.2.3 解耦 Anchor-Free 检测头（Head）

YOLOv8 检测头最重要的改进是**完全抛弃了传统的 Anchor（锚框）机制**，改为 **Anchor-Free** 设计，并将分类分支与定位分支完全解耦：

**传统锚框 YOLOv5 的局限**：锚框需要对训练数据进行 K-means 聚类预设，域外分布下聚类结果可能不适配，且回归目标为相对锚框的偏移量，增加了学习难度。

**YOLOv8 Anchor-Free 方案**：
- **定位分支**：对每个栅格点直接预测到边界框四边的距离分布（Distribution），即将连续的偏移量离散化为 $[0, R]$ 上的概率分布，通过 **DFL（Distribution Focal Loss）** 监督：

$$\text{DFL}(S_i, S_{i+1}) = -\left[(y_{i+1} - y)\log(S_i) + (y - y_i)\log(S_{i+1})\right]$$

其中 $y$ 为真实距离值，$S_i$、$S_{i+1}$ 为离散分布的相邻概率，DFL 鼓励概率质量集中在真实值附近的两个整数网格上，相比 $L_1/L_2$ 回归对模糊标注更鲁棒。

- **分类分支**：独立于定位分支，使用 Binary Cross-Entropy（BCE）损失对每个类别独立分类，避免了耦合检测头中分类置信度与定位质量相互干扰的问题。

**任务对齐学习（Task-Aligned Learning, TAL）**：YOLOv8 使用 TAL 进行正负样本分配，综合考虑分类得分和定位 IoU，将对齐度最高的 $\text{topk}$ 个锚点分配为正样本，避免了传统 IoU 阈值方法在边界样本上的歧义。

### 6.2.4 推理后处理

推理时，YOLOv8 对每个输出特征图的每个栅格点解码预测结果，经 Sigmoid 激活分类分数后，使用**非极大值抑制（NMS）**过滤重叠预测框。本系统推理时的 NMS 参数：

```python
CONF_THRESHOLD = 0.35   # 置信度阈值（低于此值的框直接过滤）
IOU_THRESHOLD  = 0.45   # NMS IoU 阈值（重叠度超过此值保留置信度更高的框）
```

## 6.3 自定义数据集构建

### 6.3.1 检测目标定义

本系统以**室内门的开/关状态**作为检测目标，定义两个类别：
- `closed`（关闭）：门处于完全或近似关闭状态（开度 < 20°）；
- `open`（开启）：门处于可见开度 ≥ 20° 的打开状态。

选择门开/关状态检测的理由：（1）具有代表性，门状态是楼道巡检最常见的安防检测目标之一；（2）视觉特征鲜明，门边框的几何结构在不同光照下较稳定；（3）二分类任务难度适中，适合在有限样本下验证迁移学习效果；（4）可直接迁移至配电柜门、消防栓箱门等其他设施状态检测。

### 6.3.2 数据采集策略

为保证数据集的多样性，覆盖以下维度进行拍摄：

| 维度 | 覆盖范围 |
|:---|:---|
| 拍摄距离 | 0.5m、1.0m、1.5m、2.0m（共4个距离档位） |
| 拍摄角度 | 正对门（0°）、斜对15°、斜对30°（左右各一） |
| 光照条件 | 自然光充足、自然光不足（傍晚）、走廊灯光（夜间）、逆光 |
| 门的类别 | 木质单开门（主要）、玻璃门（辅助）、防盗门（辅助） |
| 背景复杂度 | 纯色墙面背景、有杂物的走廊背景、行人经过（遮挡） |
| 开启程度（open类）| 约20°、45°、90°、全开（≥120°）各若干张 |

### 6.3.3 标注工具与标注规范

使用 **LabelImg**（Tzutalin, 2015）进行边界框标注，导出为 YOLOv8 格式（YOLO TXT）：每张图像对应一个同名 `.txt` 文件，每行格式为：

```

```

其中坐标均为相对图像宽高的归一化值（$[0, 1]$），`class_id` 为 0（`closed`）或 1（`open`）。

### 6.3.4 数据增强策略

YOLOv8 训练时默认启用以下数据增强（在 `train.py` 中通过 `augment=True` 激活，对训练集在线动态增强）：

| 增强方式 | 参数 | 目的 |
|:---|:---:|:---|
| Mosaic | 概率 1.0（最后10 epoch关闭） | 拼接4张图像，增加小目标检测能力，提升批内样本多样性 |
| HSV 颜色扰动 | H: ±0.015, S: ±0.7, V: ±0.4 | 模拟不同光照与色温条件 |
| 随机水平翻转 | 概率 0.5 | 增加左右对称的门朝向样本 |
| 随机缩放 | ±50% | 增加不同距离下的尺度多样性 |
| 随机平移 | ±10% | 增加目标在图像中不同位置的出现 |
| MixUp | 概率 0.1 | 线性混合两张图像，提升模型对边界样本的鲁棒性 |
| CopyPaste | 概率 0.1 | 将标注目标粘贴到其他背景，增加遮挡场景 |

**注意**：水平翻转对门状态检测是安全增强——翻转后门的开/关状态标签不变，视觉特征仍然保持一致。垂直翻转（`flipud`）设为0，因为翻转后的门在现实中不存在，可能引入噪声标注。

## 6.4 迁移学习与模型训练

### 6.4.1 迁移学习策略

迁移学习（Transfer Learning）在小样本自定义检测任务中至关重要。Pan 和 Yang [17] 指出，当源域（COCO 大规模检测预训练）与目标域（室内门开关状态，600张）在低级特征（边缘、纹理、形状）层面具有相似性时，直接复用预训练的低级特征提取器，仅微调高级语义理解层，可以用最少的目标域数据达到较高的检测精度。

本系统采用如下两阶段迁移学习策略：

**阶段一：主干冻结预热（Freeze Epochs，前10轮）**

冻结主干（Backbone）前10层参数，仅优化颈部（Neck）和检测头（Head），使检测头快速适应自定义类别的分类分布，防止前期主干随机梯度破坏预训练低级特征的稳定性。

**阶段二：全局微调（Fine-tuning，后110轮）**

解冻主干所有参数，以较小的全局学习率对整个网络进行联合微调，使主干深层语义特征逐步适应室内门的外观特征（区分关门/开门的颈部图案、门缝深度等细节）。

### 6.4.2 训练超参数配置

```python
# train.py 主要超参数
model.train(
    data    = 'data.yaml',
    epochs  = 120,              # 总训练轮数
    imgsz   = 640,              # 输入图像尺寸（正方形）
    batch   = 16,               # 批次大小（在训练主机上运行，非树莓派）
    workers = 4,                # 数据加载进程数
    device  = 0,                # GPU 0（训练时使用 CUDA）

    # 学习率调度
    lr0     = 0.01,             # 初始学习率
    lrf     = 0.01,             # 最终学习率 = lr0 × lrf（余弦退火）
    warmup_epochs = 3,          # 预热轮数

    # 正则化
    weight_decay = 0.0005,      # L2 正则化系数
    dropout = 0.0,              # YOLOv8 检测任务不使用 dropout

    # 迁移学习冻结
    freeze  = 10,               # 冻结主干前10层（阶段一结束后自动解冻）

    # 数据增强
    mosaic  = 1.0,              # Mosaic 增强概率
    mixup   = 0.1,              # MixUp 增强概率
    flipud  = 0.0,              # 禁止垂直翻转
    fliplr  = 0.5,              # 水平翻转概率

    # 早停与保存
    patience = 30,              # 早停耐心值（验证 mAP 连续30轮不提升则停止）
```

### 6.4.4 类别不平衡处理

若训练集中 `closed` 类别图像数量远多于 `open` 类别（如楼道大部分时间门关闭），可能导致模型偏向于预测 `closed`。处理策略：
1. **采集均衡**：确保两类样本比例在 40:60 至 60:40 之间；
2. **加权采样**：在数据加载时对少数类别图像提高采样权重（YOLOv8 可通过自定义 `Dataset` 实现）；
3. **置信度阈值调节**：若验证集上 `open` 的 Recall 偏低，适当降低 `CONF_THRESHOLD`（从0.35降至0.25）提高召回。

## 6.5 嵌入式部署与推理优化

### 6.5.1 ultralytics PyTorch 直接推理方案

本系统最终采用 **ultralytics 框架直接加载 PyTorch 权重（`.pt`）** 在树莓派5上进行 CPU 推理。该方案无需模型格式转换，直接使用训练产出的 `best.pt` 权重文件，部署流程最为简洁：

```python
from ultralytics import YOLO

# 加载训练好的模型
model = YOLO('best.pt')

# 单帧推理
results = model.predict(source=frame, conf=0.35, iou=0.45, verbose=False)
```

部署优势：
- 无需额外的 ONNX 导出步骤，训练完成后直接部署；
- ultralytics 内置完整的前/后处理流水线（输入缩放、NMS、结果解析），代码简洁可维护；
- 可通过 `model.predict()` 的 `conf`、`iou` 参数灵活调整检测阈值。

### 6.5.2 ONNX 导出（备选优化方案）

若后续需进一步提升推理速度，可将 PyTorch 权重导出为 **ONNX** 格式，以便使用 ONNXRuntime 推理引擎进行加速（无需 CUDA/TensorRT）：

```python
from ultralytics import YOLO

model = YOLO('runs/detect/door_yolov8n/weights/best.pt')
model.export(
    format    = 'onnx',
    imgsz     = 640,
    opset     = 12,         # ONNX opset 12（ONNXRuntime 1.14+ 支持）
    simplify  = True,       # 使用 onnx-simplifier 化简计算图
    dynamic   = False,      # 固定 batch=1（推理时固定输入尺寸，加速初始化）
)
# 导出结果：best.onnx（约 6.2 MB）
```

`simplify=True` 通过 onnx-simplifier 合并常量节点、消除冗余算子，通常可将推理耗时再减少 5~10%。

### 6.5.4 ONNX 推理代码实现（备选方案参考）

以下代码展示了基于 ONNXRuntime 的推理封装类实现，作为后续推理加速的备选参考方案：

```python
import numpy as np
import cv2
import onnxruntime as ort

class YOLOv8Detector:
    """YOLOv8n ONNX 推理封装类"""
    CLASS_NAMES = ['closed', 'open']

    def __init__(self, model_path: str, conf_thres=0.35, iou_thres=0.45):
        # 初始化 ONNXRuntime 推理会话（CPU，多线程）
        so = ort.SessionOptions()
        so.intra_op_num_threads = 4     # 使用4个核心（BCM2712四核全部用上）
        so.execution_mode = ort.ExecutionMode.ORT_PARALLEL
        self.session = ort.InferenceSession(
            model_path,
            sess_options=so,
            providers=['CPUExecutionProvider']
        )
        self.conf_thres = conf_thres
        self.iou_thres  = iou_thres
        # 从模型元信息获取输入尺寸
        inp = self.session.get_inputs()[0]
        self.input_name = inp.name
        _, _, self.h, self.w = inp.shape   # (1, 3, 640, 640)

    def preprocess(self, img_bgr: np.ndarray) -> np.ndarray:
        """BGR图像 → NCHW float32（归一化到[0,1]）"""
        img = cv2.resize(img_bgr, (self.w, self.h))
        img = cv2.cvtColor(img, cv2.COLOR_BGR2RGB)
        img = img.astype(np.float32) / 255.0
        return img.transpose(2, 0, 1)[np.newaxis]  # (1, 3, H, W)

    def postprocess(self, output: np.ndarray,
                    orig_h: int, orig_w: int) -> list:
        """解码 YOLOv8 输出张量 (1, 6, 8400) → 检测结果列表"""
        preds = output[0].T                         # (8400, 6)
        # 过滤置信度
        scores = preds[:, 4:]                       # (8400, nc)
        class_ids = scores.argmax(axis=1)
        confidences = scores[np.arange(len(scores)), class_ids]
        mask = confidences > self.conf_thres
        preds = preds[mask]
        class_ids = class_ids[mask]
        confidences = confidences[mask]
        if len(preds) == 0:
            return []
        # 解码 xywh → xyxy（原始图像坐标）
        boxes_xywh = preds[:, :4]
        scale_x, scale_y = orig_w / self.w, orig_h / self.h
        x1 = (boxes_xywh[:, 0] - boxes_xywh[:, 2] / 2) * scale_x
        y1 = (boxes_xywh[:, 1] - boxes_xywh[:, 3] / 2) * scale_y
        x2 = (boxes_xywh[:, 0] + boxes_xywh[:, 2] / 2) * scale_x
        y2 = (boxes_xywh[:, 1] + boxes_xywh[:, 3] / 2) * scale_y
        boxes = np.stack([x1, y1, x2, y2], axis=1)
        # NMS
        indices = cv2.dnn.NMSBoxes(
            boxes.tolist(), confidences.tolist(),
            self.conf_thres, self.iou_thres
        )
        results = []
        for i in (indices.flatten() if len(indices) > 0 else []):
            results.append({
                'label': self.CLASS_NAMES[class_ids[i]],
                'conf':  float(confidences[i]),
                'bbox':  boxes[i].tolist()
            })
        return results

    def detect(self, img_bgr: np.ndarray) -> list:
        """完整推理流程：预处理 → 前向推理 → 后处理"""
        h, w = img_bgr.shape[:2]
        inp = self.preprocess(img_bgr)
        out = self.session.run(None, {self.input_name: inp})
        return self.postprocess(out[0], h, w)
```

## 6.6 检测 ROS2 节点封装

### 6.6.1 节点设计原则

视觉检测节点采用 **连续订阅推理 + `max_fps` 参数节流** 模式。节点持续订阅摄像头图像话题（`/image_raw`），每收到一帧图像即执行一次 YOLOv8n 推理，但通过 `max_fps` 参数限制最大推理频率（默认 5.0 FPS），避免过度占用 CPU 资源。设计理由：巡检场景下机器人移动速度较慢（约 0.15–0.25 m/s），目标状态变化频率低，5 FPS 的检测频率已完全满足实时性需求；同时连续推理可在巡检移动过程中持续监控环境，而非仅在到达巡检点时触发单次检测。

节点将检测结果以 `vision_msgs/Detection2DArray` 消息格式发布到 `/yolo/detections` 话题，同时发布带标注框的可视化图像到 `/yolo/annotated` 话题，供 `inspection_manager` 节点订阅进行结果记录与图像保存。

### 6.6.2 ROS2 检测节点核心实现

```python
#!/usr/bin/env python3
# yolo_detector.py——核心推理逻辑简化版
import rclpy
from rclpy.node import Node
from sensor_msgs.msg import Image
from vision_msgs.msg import Detection2DArray, Detection2D, ObjectHypothesisWithPose
from cv_bridge import CvBridge
from ultralytics import YOLO
import time

class YoloDetectorNode(Node):
    def __init__(self):
        super().__init__('yolo_detector')
        # 声明参数
        self.declare_parameter('model_path', 'best.pt')
        self.declare_parameter('conf_thres', 0.35)
        self.declare_parameter('max_fps', 5.0)
        self.declare_parameter('image_topic', '/image_raw')

        model_path = self.get_parameter('model_path').value
        self.conf = self.get_parameter('conf_thres').value
        self.max_fps = self.get_parameter('max_fps').value
        image_topic = self.get_parameter('image_topic').value

        # 加载 YOLOv8n 模型（ultralytics PyTorch）
        self.model = YOLO(model_path)
        self.bridge = CvBridge()
        self.last_infer_time = 0.0

        # 订阅图像话题
        self.sub = self.create_subscription(
            Image, image_topic, self._image_cb, 10)
        # 发布检测结果与可视化图像
        self.det_pub = self.create_publisher(
            Detection2DArray, '/yolo/detections', 10)
        self.ann_pub = self.create_publisher(
            Image, '/yolo/annotated', 10)

        self.get_logger().info(
            f'YOLOv8n 检测节点启动，模型：{model_path}，'
            f'max_fps={self.max_fps}')

    def _image_cb(self, msg: Image):
        # max_fps 节流
        now = time.time()
        if now - self.last_infer_time < 1.0 / self.max_fps:
            return
        self.last_infer_time = now

        frame = self.bridge.imgmsg_to_cv2(msg, 'bgr8')
        results = self.model.predict(
            source=frame, conf=self.conf, verbose=False)

        # 封装 Detection2DArray 并发布
        det_msg = Detection2DArray()
        det_msg.header = msg.header
        for r in results:
            for box in r.boxes:
                d = Detection2D()
                hyp = ObjectHypothesisWithPose()
                hyp.hypothesis.class_id = r.names[int(box.cls)]
                hyp.hypothesis.score = float(box.conf)
                d.results.append(hyp)
                det_msg.detections.append(d)
        self.det_pub.publish(det_msg)

        # 发布带标注框的可视化图像
        annotated = results[0].plot()
        ann_msg = self.bridge.cv2_to_imgmsg(annotated, 'bgr8')
        ann_msg.header = msg.header
        self.ann_pub.publish(ann_msg)


def main(args=None):
    rclpy.init(args=args)
    node = YoloDetectorNode()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```
