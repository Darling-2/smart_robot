# 第4章 SLAM建图与AMCL定位（GMapping主线，Cartographer扩展对比）

## 4.1 激光SLAM技术概述

同步定位与建图（SLAM，Simultaneous Localization and Mapping）是移动机器人自主导航的核心基础问题，要求机器人在**无先验地图**的未知环境中，仅凭自身携带的传感器观测数据，同时估计自身运动轨迹（定位）和构建环境地图（建图）。由于定位依赖地图、建图又依赖定位，两者相互耦合，是机器人学中最具挑战性的基础问题之一。

从概率论角度，SLAM 的目标是估计机器人轨迹 $\mathbf{x}_{1:t}$ 和环境地图 $\mathbf{m}$ 在传感器观测 $\mathbf{z}_{1:t}$ 和控制输入 $\mathbf{u}_{1:t}$ 条件下的后验概率：

$$p\!\left(\mathbf{x}_{1:t},\, \mathbf{m} \mid \mathbf{z}_{1:t},\, \mathbf{u}_{1:t}\right) \quad \text{（式4-1）}$$

由于精确求解此后验分布在高维空间中计算上不可行，实用 SLAM 算法均采用近似策略。对于本科阶段的工程实践，理解该公式的核心含义即可：**机器人根据传感器观测数据不断更新对自身位置和环境地图的"最优猜测"**。

按照求解方法可将激光 SLAM 分为两大类：

**滤波器方法**以扩展卡尔曼滤波（EKF-SLAM）和粒子滤波（FastSLAM、GMapping [10]）为代表，采用递推方式在线更新位姿与地图的联合概率分布。GMapping 利用 Rao-Blackwellized 粒子滤波将 SLAM 分解为机器人轨迹的粒子集表示与各粒子对应的条件独立地图，计算复杂度相对可控，是 ROS1 社区最广泛使用的二维激光 SLAM 方案。然而，GMapping **不具备显式的回环检测机制**，在走廊等重复性高的场景中长距离行驶后会产生不可修正的累积误差，导致建图结果出现"鬼影"或连通路径断裂等问题。

**图优化方法**以 Cartographer [8]、RTAB-Map 等为代表，显式维护节点（机器人位姿）与约束（传感器观测匹配关系）构成的**位姿图（Pose Graph）**，通过批量非线性优化修正全局一致性，并集成专用的**回环检测模块**用于识别历史访问区域并引入长程约束。图优化方法的全局一致性明显优于滤波器方法，代价是需要在回环检测时执行一次全局优化，计算量较大但并不需要实时执行。

在本课题的最终集成方案中，考虑到双板分工与工程部署边界（香橙派侧运行 ROS1 导航闭环，树莓派侧运行 ROS2 视觉与任务调度），SLAM 建图选择 **ROS1 `slam_gmapping`（GMapping）** 作为主线实现，满足走廊场景的二维建图需求；**Google Cartographer** 的原理与参数配置作为扩展对比内容保留，用于展示图优化 SLAM 的回环检测优势与后续迁移可能性。Cartographer 的整体处理流程如图4-1所示。

## 4.2 Cartographer 2D SLAM 算法原理（扩展对比）

### 4.2.1 系统架构概述

Cartographer 的处理流水线分为**局部 SLAM**（Local SLAM）和**全局 SLAM**（Global SLAM）两个并行运行的模块：

- **局部 SLAM**：以每次新到达的激光扫描帧为输入，通过扫描匹配（Scan Matching）估计机器人相对于**当前子图（Submap）**的位姿增量，实时维护当前子图的占据概率，直到子图"满"后关闭并开启新子图；
- **全局 SLAM**：在后台异步运行，持续检测新关闭的子图与历史子图的重叠（回环），通过分支定界搜索提出候选约束，并通过 Ceres Solver 求解全局位姿图优化问题，将修正量分发给所有历史节点，消除全局漂移。

两个模块的分工使得 Cartographer 能够在单核 CPU 上实时运行局部 SLAM（10Hz 激光帧），同时以异步低优先级执行代价较高的回环检测，兼顾实时性与全局一致性。

### 4.2.2 局部扫描匹配

**子图表示**：每个子图是一个小型的二维占据概率栅格（Occupancy Grid），分辨率与全局地图一致（本系统 0.05m/格）。子图维护每格的占据概率 $M(x,y) \in [0,1]$，通过贝叶斯更新将激光测量点（占据）与透射路径（空闲）纳入概率估计。为便于优化，在梯度下降求解前使用**双三次插值**将离散概率栅格 $M$ 连续化为光滑函数 $M_{\text{smooth}}$。

**扫描匹配优化**：给定当前子图与一帧新到的激光扫描（$K$ 个测量点 $h_k$），局部 SLAM 求解位姿增量 $\xi = (t_x, t_y, \theta)$，使所有测量点在变换后尽可能落在子图中的已占据区域。优化目标函数为：

$$\xi^* = \arg\min_{\xi} \sum_{k=1}^{K} \left[1 - M_{\text{smooth}}\!\left(T_{\xi} \cdot h_k\right)\right]^2$$

其中 $T_{\xi}$ 为由位姿增量 $\xi$ 构成的刚体变换矩阵，将每个测量点从激光雷达坐标系变换到子图坐标系。当 $M_{\text{smooth}}(T_\xi h_k) = 1$ 时（点落在已知障碍物上），该点对目标函数贡献为零；当点落在空闲区域时贡献正值。通过 **Gauss-Newton 法**（高斯牛顿非线性最小二乘迭代）求解上述优化问题，收敛后得到精确的局部位姿估计。

初始位姿估计由里程计（`/odom`）预测提供，作为 Gauss-Newton 迭代的初始点，保证迭代收敛至全局最优（局部线性化范围内）。对于里程计累积误差较大的情况，Cartographer 还在 Gauss-Newton 之前执行一次**分支定界粗搜索**（Coarse-to-Fine，仅限局部搜索窗口），进一步提高匹配鲁棒性。

### 4.2.3 回环检测与全局位姿图优化

**回环检测**：当一个新子图关闭后，Cartographer 尝试将其与所有历史子图进行匹配，寻找回环约束。搜索策略采用**分支定界法（Branch-and-Bound）**：在预先构建的多分辨率占据地图上，递归地将候选位姿空间分割为子区域，利用各子区域内得分的上界剪枝，高效搜索高匹配分数的候选变换，而无需穷举所有位姿候选。

对于通过分数阈值（本系统配置 0.45）验证的回环候选，构建**位姿约束** $\xi_{ij}$（子图 $i$ 与节点 $j$ 之间的相对变换估计）及其协方差矩阵 $\Sigma_{ij}$，加入全局位姿图。

**全局位姿图优化**：全局 SLAM 以如下稀疏非线性最小二乘问题为优化目标：

$$\min_{\Xi^m, \Xi^s}\ \frac{1}{2} \sum_{ij} \rho\!\left(\mathbf{E}^2\!\left(\xi_i^m,\, \xi_j^s;\, \Sigma_{ij},\, \xi_{ij}\right)\right)$$

其中 $\Xi^m = \{\xi_i^m\}$ 为所有子图位姿，$\Xi^s = \{\xi_j^s\}$ 为所有激光扫描节点位姿，$\mathbf{E}(\cdot)$ 为位姿误差函数，$\rho(\cdot)$ 为 Huber 鲁棒核函数（减弱错误回环约束的影响）。此优化问题由 **Ceres Solver**（Google 开源的非线性最小二乘库）的稀疏 Levenberg-Marquardt 求解器高效求解，最终输出全局一致的子图位姿与扫描节点位姿。

### 4.2.4 `bingda_lds_2d.lua` 关键参数配置

在扩展对比方案中，使用 `robot_navigation_ros2/config/bingda_lds_2d.lua` 配置文件（基于 Cartographer 官方 `backpack_2d.lua` 针对 EAI 单线雷达优化），关键参数说明如下：

```lua
-- bingda_lds_2d.lua 关键参数（节选）
options = {
  map_builder = MAP_BUILDER,
  trajectory_builder = TRAJECTORY_BUILDER,
}

MAP_BUILDER.use_trajectory_builder_2d = true

-- 子图参数
TRAJECTORY_BUILDER_2D.submaps.num_range_data = 35  -- 每子图累积帧数，影响子图大小
TRAJECTORY_BUILDER_2D.submaps.grid_options_2d.resolution = 0.05  -- 地图分辨率 5cm

-- 扫描匹配参数
TRAJECTORY_BUILDER_2D.use_online_correlative_scan_matching = true  -- 在线粗搜索增强鲁棒性
TRAJECTORY_BUILDER_2D.real_time_correlative_scan_matcher.linear_search_window = 0.1
TRAJECTORY_BUILDER_2D.real_time_correlative_scan_matcher.angular_search_window = math.rad(20.)

-- 运动滤波（避免静止时重复加入相同帧）
TRAJECTORY_BUILDER_2D.motion_filter.max_distance_meters = 0.2
TRAJECTORY_BUILDER_2D.motion_filter.max_angle_radians = math.rad(1.0)

-- 回环检测参数
POSE_GRAPH.optimize_every_n_nodes = 20  -- 每20个新节点触发一次全局优化
POSE_GRAPH.constraint_builder.min_score = 0.45  -- 降低阈值以应对走廊重复特征
POSE_GRAPH.constraint_builder.global_localization_min_score = 0.6
```

`min_score` 从默认 0.55 降低至 0.45 是针对走廊环境的重要调整——走廊两侧平行墙体在单线激光下特征单一，默认阈值下回环候选得分普遍偏低，导致回环无法触发；适当降低阈值后回环检测率显著提升，以少量误检增加 Huber 鲁棒核承受的代价，整体建图质量更高。

## 4.3 EAI激光雷达驱动配置

### 4.3.1 驱动节点配置

香橙派3LTS 上运行 `ydlidar_ros_driver`（ROS1 Noetic 版本），通过 USB 串口（`/dev/ydlidar`，波特率115200）采集 EAI 激光雷达数据。对应 launch 文件配置如下：

```python
# lidar_ros2/ydlidar_ros2_driver/launch/ 参考（ROS1 对应配置类似）
ydlidar_params = {
    'port': '/dev/ydlidar',
    'baudrate': 115200,
    'frame_id': 'laser_link',      # 发布话题的坐标系名称
    'ignore_array': '',
    'resolution_fixed': True,
    'auto_reconnect': True,
    'reversion': False,
    'angle_min': -180.0,           # 扫描角度范围：全360°
    'angle_max': 180.0,
    'range_min': 0.15,             # 最小测距 15cm（盲区）
    'range_max': 6.0,              # 最大测距 6m
    'frequency': 10.0,             # 扫描频率 10Hz
    'invalid_range_is_inf': False,
    'sampleRate': 3,               # 采样率（kHz）
}
```

设备权限配置：通过 udev 规则（`script/lidar_udev.sh`）将 YDLidar 设备固定映射到 `/dev/ydlidar`，避免设备号因 USB 端口变化而漂移：

```bash
# /etc/udev/rules.d/ydlidar.rules
KERNEL=="ttyUSB*", ATTRS{idVendor}=="10c4", ATTRS{idProduct}=="ea60", \
  SYMLINK+="ydlidar", MODE="0666"
```

### 4.3.2 TF 坐标变换配置

激光雷达坐标系 `laser_link` 相对于底盘中心 `base_link` 的静态 TF 变换在 URDF 中定义（位于 `robot_description/urdf/nanorobot_description.urdf`）。`robot_state_publisher` 节点解析 URDF 并发布 `/tf_static`，用于在香橙派 ROS1 域内保证 `slam_gmapping`、AMCL 与 `move_base` 的 TF 链路一致。

EAI 激光雷达发布的 `/scan` 话题帧率约 10Hz，每帧包含约 360 个测量点（1° 角分辨率），消息类型为 `sensor_msgs/LaserScan`，包含 `range_min`（0.15m）、`range_max`（6.0m）等字段。对于扩展对比方案中的 Cartographer，会在扫描匹配前自动过滤范围外的无效测量点（`inf` 值或 0）。

## 4.4 建图实验与结果分析

### 4.4.1 实验环境与过程

**实验环境**：公寓楼道，场地面积约 100m²，典型格局包含一条主走廊（长约 20m、宽约 1.5m）、两个T形交叉路口和数个门洞。走廊两侧主要为墙壁、门框和少量杂物，特征较为单一，是对激光 SLAM 算法具有挑战性的典型室内场景。

**建图过程**：按以下步骤完成地图构建：

1. 启动香橙派端底盘控制与激光雷达驱动；
2. 在香橙派（ROS1）端启动 `slam_gmapping` 建图；
3. 启动 RViz（ROS1）实时监控建图进度（`/map` 与 TF 可视化）；
4. 通过蓝牙遥控器或 `teleop_twist_keyboard` 以约 0.10~0.15m/s 的速度驱动机器人沿走廊缓慢行进一圈（单圈约 5~8 分钟），确保走廊各区域均被充分覆盖；
5. 回到起始位置后，确认 RViz 中地图边界闭合良好；
6. 执行地图保存命令（ROS1）：
  ```bash
  rosrun map_server map_saver -f ~/map
  ```
  生成 `map.pgm`（灰度图像）和 `map.yaml`（分辨率、原点、阈值等元信息）。

说明：Cartographer 建图与回环修正作为扩展对比可在树莓派5（ROS2）侧单独启动验证，用于展示图优化SLAM的优势与迁移可能性。

### 4.4.3 参数调优经验

在实际建图实验中，遇到以下问题并总结了相应调优经验。

**（主线）GMapping 调参要点**：
- 将地图分辨率设为 0.05m，在走廊边界清晰度与资源占用之间取得折中；
- 通过适当降低建图速度（约 0.10~0.15m/s）减少里程计噪声对粒子滤波的扰动；
- 结合走廊场景调节 `linearUpdate`/`angularUpdate` 等更新阈值，避免过于频繁的地图更新导致抖动，同时保证拐角处的特征能被及时纳入地图。

**（扩展对比）Cartographer 调参经验**：

**问题1——走廊漂移（回环未触发）**：机器人沿长走廊行进约 15m 后，地图出现明显侧向偏移，但回环未触发修正。**解决方案**：将 `min_score` 从 0.55 降低至 0.45，同时将 `optimize_every_n_nodes` 从 90 降低至 20，提高回环检测灵敏度和优化频率，问题解决。

**问题2——建图抖动（里程计噪声）**：在地板砖缝隙区域驱动时，编码器读数产生跳变，导致子图位姿短暂抖动。**解决方案**：适当增大运动滤波阈值（`max_distance_meters: 0.2`），过滤过小的位移更新，同时降低建图速度至 0.1~0.15m/s。

**问题3——地图分辨率与文件大小**：初始使用 0.025m 分辨率，地图文件 > 5MB，加载时内存占用较高。**解决方案**：恢复至 0.05m 分辨率，文件大小降至约 1.5MB，导航栈（如 `move_base` 的代价地图与规划器）在 5cm 分辨率下已完全满足室内走廊巡检需求。

## 4.5 AMCL 自主定位原理与配置

### 4.5.1 自适应蒙特卡洛定位原理

地图构建完成后，系统切换至 **AMCL（Adaptive Monte Carlo Localization，自适应蒙特卡洛定位）** 进行实时自主定位。AMCL 属于粒子滤波方法，通过维护一个粒子集 $\mathcal{X}_t = \{(\mathbf{x}_t^{[i]}, w_t^{[i]})\}_{i=1}^{N}$ 来近似表示机器人位姿 $\mathbf{x}_t = (x, y, \theta)$ 的后验概率分布，其中 $\mathbf{x}_t^{[i]}$ 为第 $i$ 个粒子的位姿假设，$w_t^{[i]}$ 为对应的权重。

粒子滤波的递推步骤遵循贝叶斯滤波公式：

$$bel(\mathbf{x}_t) = \eta \cdot p\!\left(\mathbf{z}_t \mid \mathbf{x}_t\right) \int p\!\left(\mathbf{x}_t \mid \mathbf{u}_t,\, \mathbf{x}_{t-1}\right)\, bel(\mathbf{x}_{t-1})\, d\mathbf{x}_{t-1}$$

在实现中分为三步：

**步骤1——运动预测（Motion Update）**：根据里程计增量 $\mathbf{u}_t$（`/odom` 话题），用**差速驱动运动模型**（含噪声参数 $\alpha_1 \sim \alpha_4$）对每个粒子进行位姿采样，将粒子集从时刻 $t-1$ 传播至时刻 $t$ 的预测位姿：

$$\mathbf{x}_t^{[i]} \sim p\!\left(\mathbf{x}_t \mid \mathbf{u}_t,\, \mathbf{x}_{t-1}^{[i]}\right)$$

**步骤2——观测更新（Measurement Update）**：根据当前激光观测 $\mathbf{z}_t$（`/scan` 话题），结合已知地图 $\mathbf{m}$，计算每个粒子的观测似然并更新权重：

$$w_t^{[i]} = p\!\left(\mathbf{z}_t \mid \mathbf{x}_t^{[i]},\, \mathbf{m}\right)$$

本系统配置 `likelihood_field` 激光观测模型（`NanoRobot.yaml: laser_model_type: likelihood_field`）：对每个激光测量点，查询其在地图中到最近障碍物的距离 $d_k$，通过高斯函数 $\mathcal{N}(d_k;\, 0,\, \sigma^2)$ 计算单点似然，全帧似然为各点乘积的对数和，计算效率远高于光束模型（Beam Model）。

**步骤3——重采样（Resampling）**：按归一化权重对粒子集进行有放回采样，权重高的粒子（与观测吻合好）被复制，权重低的粒子被淘汰，使粒子集向后验分布集中。

### 4.5.2 自适应粒子数（KLD 采样）

标准粒子滤波使用固定粒子数 $N$，当定位不确定性高时粒子不足导致漏检，定位收敛后粒子冗余浪费计算资源。AMCL 采用 **KLD（Kullback-Leibler Divergence）采样**策略，根据当前粒子集对状态空间的覆盖密度，动态估计满足精度要求所需的最小粒子数：

$$N_{\text{KLD}} = \frac{k-1}{2\varepsilon} \chi^2_{k-1,\,1-\delta}$$

其中 $k$ 为粒子集覆盖的非空直方图格数，$\varepsilon$ 为允许的 KL 散度误差，$\delta$ 为置信度参数。本系统中（`NanoRobot.yaml`）：

```yaml
# AMCL 粒子数自适应配置
min_particles: 200    # 定位收敛后最少保留粒子数
max_particles: 1000   # 初始化/重定位时最多粒子数
pf_err: 0.05          # KLD ε 参数
pf_z: 0.99            # KLD δ 参数（置信度）
```

实际运行中，机器人初始化时粒子数约为 800~1000（位姿不确定性大），定位收敛后（约5秒）降至 200~300，显著降低树莓派5的 CPU 负载。

### 4.5.3 关键 AMCL 参数配置

结合 `NanoRobot.yaml` 中的完整 AMCL 参数，关键配置项说明如下：

```yaml

  ros__parameters:
    # 激光观测模型
    laser_model_type: "likelihood_field"    # 概率场模型（效率高）

    laser_min_range: 0.15
    max_beams: 60                           # 每帧采用的激光射线数（降采样，降低计算量）
    laser_likelihood_max_dist: 2.0          # 地图距离查询最大范围

    alpha1: 0.2    # 旋转对旋转的噪声
    alpha2: 0.2    # 平移对旋转的噪声
    alpha3: 0.2    # 平移对平移的噪声
    alpha4: 0.2    # 旋转对平移的噪声

    # 重采样参数

    recovery_alpha_fast: 0.1    # 快速平均权重滤波器参数

    tf_broadcast: true
```

本章系统阐述了本系统 SLAM 建图与实时定位模块的算法原理与工程实现。在建图方面：介绍了激光 SLAM 的概率建模框架，对比了滤波器方法（以 GMapping 为代表）与图优化方法（以 Cartographer 为代表）的优劣；最终集成主线采用 ROS1 `slam_gmapping` 满足走廊二维建图需求，同时保留 Cartographer 的处理流程、扫描匹配与位姿图优化原理及关键参数作为扩展对比材料。在定位方面：推导了 AMCL 粒子滤波三步递推（运动预测→观测更新→重采样），分析了 KLD 自适应粒子数机制（200~1000 粒子动态调整），并给出关键参数说明；定位实验表明稳定行走后平移误差满足 ≤10cm 的设计指标，具备一定的干扰恢复能力。建图产生的 `map.pgm`/`map.yaml` 与 AMCL 的实时位姿估计为后续 move_base 导航模块提供必要的先验地图与实时状态反馈。
