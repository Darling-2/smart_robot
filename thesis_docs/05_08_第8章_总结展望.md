# 第8章 总结与展望

## 8.1 工作总结

本文围绕"低成本嵌入式双板异构平台上的室内自主巡检机器人"这一主题，以**香橙派3LTS（ROS1 Noetic，底盘与导航侧）**和**树莓派5（ROS2 Jazzy，视觉与任务侧）**为硬件载体，系统地研究并实现了从底层驱动到高层任务决策的完整机器人软件栈，完成了 SLAM 建图、自主导航、目标状态检测和巡检任务管理四大功能模块的设计、实现与综合实验验证。主要研究工作和贡献总结如下：

**（1）异构双板跨版本通信架构设计**

提出并实现了基于**百兆以太网直连**的 ROS1/ROS2 异构跨板通信方法。针对树莓派5（Ubuntu 24.04 + ROS2 Jazzy）不引入 ROS1 依赖的部署约束，系统采用 **ZeroMQ（ZMQ）应用层业务消息**替代 ROS Topic 级透明桥接：树莓派侧仅下发导航目标（`nav_goal`），香橙派侧回传导航结果（`nav_result`），跨板接口与导航闭环解耦。实测跨板消息往返延迟（RTT）约 **X ms**（目标 ≤20 ms），消息丢失率约 **X%**（目标 ≤1%），满足巡检任务调度需求。

**（2）基于GMapping的二维激光SLAM建图**

在香橙派 ROS1 Noetic 上部署 `slam_gmapping` 完成二维栅格地图构建，结合单线激光雷达数据在走廊场景生成可用于导航的静态地图（分辨率 0.05 m/pixel），为后续定位与路径规划提供环境先验。

**（3）基于AMCL的激光蒙特卡洛定位**

在预建静态地图上部署 AMCL（Adaptive Monte Carlo Localization），通过粒子滤波对机器人位姿进行实时估计，配置粒子数 200~1000、似然场（Likelihood Field）传感器模型，实测定位均方根误差约 **X.X cm**（目标 ≤10 cm），冷启动收敛时间约 **X 秒**。

**（4）基于move_base的自主导航与障碍规避**

以 ROS1 `move_base` 框架为导航核心，集成全局路径规划与局部避障控制（可选 DWA/TEB 等局部规划器），并针对走廊狭窄空间配置机器人足迹多边形（`[[-0.09,-0.1],[0.04,-0.1],[0.04,0.1],[-0.09,0.1]]`）与膨胀半径参数，实现了巡检点间的稳健导航与避障。树莓派侧按巡检点序列下发目标并接收回执，实现巡检任务闭环。

**（5）基于YOLOv8n的设施状态视觉检测**

自建室内门开/关状态数据集（≥600张，多角度/多距离/多光照），在 COCO 预训练权重基础上以两阶段迁移学习（冻结主干前10层预热 + 全局微调）训练 YOLOv8n，在树莓派5上基于 ultralytics 框架直接加载 PyTorch 权重（`.pt`）进行 CPU 推理，配合 `max_fps` 参数将检测频率控制在约 **5 FPS**，内存占用约 **280 MB**，验证集 mAP@0.5 达到 **XX.X%**（目标 ≥80%）。检测节点采用连续订阅推理模式，通过 `max_fps` 节流避免过度占用 CPU 资源，同时保证巡检过程中的持续监控能力。

**（6）系统集成与综合实验验证**

在实验室走廊场景下设计并执行了三种难度的综合巡检实验（标准/静态障碍/动态行人，各5次），系统资源占用分析显示树莓派5（视觉与任务侧）总内存占用约 **470 MB**（占4GB RAM的约12%），待机 CPU 负载较低、推理仅在到达巡检点时短时触发。全面验证了各模块在实际硬件上的集成可行性与功能完备性。

## 8.2 创新点

本文的主要创新点归纳如下：

1. **ROS1/ROS2 异构双板协同架构**：提出并实现了低成本嵌入式平台上 ROS1 导航闭环（底盘/雷达/建图定位/规划控制）与 ROS2 视觉与任务调度的跨版本协同方案，为受限于单一计算平台算力的小型机器人提供了一种可行的异构扩展思路；

2. **轻量视觉检测与任务调度的 CPU 协同策略**：采用连续订阅推理加 `max_fps` 参数节流的检测节点设计范式，使 YOLOv8n（约 3–5 FPS，约 280 MB）在保证持续监控能力的同时避免过度占用 CPU；同时通过 ZMQ 业务消息将“导航目标/结果”与视觉任务解耦，树莓派5侧负载稳定、无需 GPU 加速，成本优势显著；

3. **面向走廊场景的 move_base 参数工程化调优**：针对走廊狭窄直线型拓扑的特点，系统化地完成了足迹多边形、膨胀半径与局部规划器（如 DWA/TEB）关键参数的联合调优，形成了可复用的走廊导航参数配置方法。

（注：Nav2/Cartographer 等 ROS2 侧导航方案作为扩展研究与对比项，可在后续工作中进一步验证与迁移。）

## 8.3 未来工作展望

本文研究工作尚存在若干不足，未来可从以下方向进一步深入：

**（1）持续建图与动态地图更新**

当前 AMCL 依赖离线预建静态地图，对环境变化适应能力有限。未来可引入 **SLAM Toolbox** 的 **lifelong mapping** 模式，在导航的同时持续更新地图，检测并融入新增障碍物（如临时摆放的设备）到静态地图层，提升长期部署的鲁棒性。

**（2）社会力导航模型**

将机器人在人员活动区域中的行为从"避障"升级为"符合社会规范的协同通行"，引入**社会力模型（Social Force Model, SFM）** [11] 或基于深度强化学习的社会导航策略，使机器人能预测行人运动意图并选择不干扰行人的绕行路径，提升人机共融走廊环境下的导航体验。

**（3）多目标视觉检测扩展**

当前 YOLOv8n 模型仅检测"门开/关"两类目标。未来可扩展检测类别至消防栓箱门、配电柜状态、地面水渍检测、紧急出口指示牌可见性等，构建更完整的室内设施安全巡检知识库。同时，可探索将检测结果结合时间戳和位置坐标自动生成巡检报告（JSON/PDF格式）。

**（4）三维障碍物感知**

引入**深度摄像头**（如 OAK-D、Intel RealSense D435i）或**3D 激光雷达**替代或补充当前的二维激光雷达，实现对门槛、电缆、低矮障碍物的三维感知，消除二维雷达盲区，提升在复杂地面环境下的通行安全性。

**（5）ONNX 导出与 INT8 量化推理加速**

当前 YOLOv8n 使用 ultralytics 框架直接加载 PyTorch 权重进行 CPU 推理，单帧耗时约 200–400 ms。未来可将模型导出为 **ONNX** 格式并采用 ONNXRuntime 推理引擎，预计可将推理耗时降至约 50–70 ms（约 15–18 FPS）；进一步通过 **INT8 量化校准**（使用验证集约 200 张图像进行校准），在接受轻微精度损失（通常 mAP@0.5 下降 1–2%）的条件下将推理速度进一步提升，为系统预留更多 CPU 资源用于其他任务。

**（6）多机协同巡检**

在算法层面扩展为多机器人系统，利用 ROS2 多域（Domain ID 隔离）和集中式任务调度服务器，将巡检走廊分段分配给多台机器人并行执行，大幅缩短大型建筑的全覆盖巡检周期，并实现机器人故障的互相备份。
